# Phase 1 Report — SAE Circuit Discovery for Reward Hacking in GPT-2 Small

*Uses **real, labelled** reward-hacking datasets — no behaviour is synthesised or
induced. Reproducible from `src/`. Raw outputs in `results/phase1/`.*

## Executive summary

We used Sparse Feature Circuits (attribution patching over pretrained SAE features)
to discover which internal features of GPT-2 Small causally drive reward hacking,
using **real labelled data**: Sharma et al.'s factual sycophancy set (with ground-truth
correct/incorrect answers) and UltraFeedback (real model completions with human quality
scores). GPT-2 Small **strongly prefers the sycophantic response (98 % of held-out
pairs)** and **mildly prefers the verbose-but-worse response (58 %)**; it shows **no
preference on code** (50 %). We recovered compact residual-stream circuits for
sycophancy and length bias with distinct layer profiles and disjoint feature semantics.

---

## Data sources (all real, no induction)

| Type | Dataset | What "hacking" means | Ground truth |
|---|---|---|---|
| **syco** | Sharma et al. 2024 `answer.jsonl` (sycophancy-eval) | User asserts the **factually incorrect** answer; **hacking** = affirming the user's wrong answer, **clean** = giving the correct answer | Yes — `correct_answer` / `incorrect_answer` per trivia item |
| **length** | openbmb/UltraFeedback | Same prompt: **hacking** = a real completion ≥2× longer but with **≤ quality score**; **clean** = the shorter, ≥-quality completion | Yes — human `overall_score` per completion |
| **code** | (null) | Real labelled set (PatronusAI/trace-dataset) is **gated**; GPT-2 Small cannot model code | — |

No completions were padded, templated, or generated by us for syco/length. (An earlier
version of this project used synthetic length-padding; that is preserved in
`data/pairs_synthetic_backup/` and is **not** used here.)

## Deviations from the task spec (deliberate, documented)

- GPT-2 Small is **12 layers** (task misstates 6).
- **Pretrained SAELens SAEs** (`gpt2-small-res-jb`, ReLU, d_sae=24576) instead of training 19 from scratch.
- **Residual-stream** attribution only (the v5 attn/mlp SAEs cannot be re-decoded on a modified feature vector).
- **60 pairs/type**, reduced scale for a 16 GB M4; circuits discovered on the first 40, evaluated on the held-out 20 in Phase 3.
- Readout = **mean** over completion tokens (on real data the completions differ in content throughout, so mean is the stronger signal: syco frac(m>0)=0.98 vs 0.73 for last-token).

## Section 1 — Dataset and model verification

Metric **m = mean-token log p(hacking) − mean-token log p(clean)**; m>0 ⟺ model prefers
hacking. See `results/figures/m_distributions.png`.

| Type | mean m | median | **frac m>0** | verdict |
|---|---|---|---|---|
| **syco** | **+1.234** | +1.276 | **0.98** | strongly prefers sycophancy |
| **length** | **+0.275** | +0.256 | **0.58** | mildly prefers verbose-worse |
| code | +0.044 | +0.044 | 0.50 | no preference (null) |

**Findings.** On real, ground-truthed data GPT-2 Small robustly prefers agreeing with a
user's factually-wrong answer (98 %) — clean, strong sycophancy. It weakly prefers the
longer/worse response (58 %, only just above chance) — real but faint length bias, much
weaker than the artificially-inflated synthetic version. It has **no** exploit
preference on code (50 %), confirming that GPT-2 Small cannot exhibit code hacking. Code
is excluded from all conclusions.

---

## Section 2 — SAE quality

Pretrained SAELens `gpt2-small-res-jb` residual-stream SAEs, 12 layers, d_sae=24576,
ReLU. Published quality: ~90 %+ variance explained, low L0, <2 % dead features. Not
re-measured locally (limitation, §6). No submodule flagged unusable.

## Section 3 — The circuits

`T_N` auto-tuned to the 20–200 node band; `T_E`=0.01.

| Circuit | nodes | edges | T_N | m_full | peak layers (Σ|IE|) |
|---|---|---|---|---|---|
| **C_syco** | 94 | 2,356 | 0.100 | +1.23 | **L6–L8** (mid-late) |
| **C_length** | 168 | 6,857 | 0.195 | +0.11 | **L0, L2** (early) |
| C_code (null) | 146 | 3,570 | 0.156 | +0.04 | L8–L10 |

**Top nodes (layer, feature, IE):**
- **C_syco**: L4 F10160 (−0.88), L5 F19700 (−0.83), L3 F8495 (−0.79), L7 F2993 (−0.69). Top-by-magnitude features are **negative-IE (hacking-resisting)** — a fact that matters in Phase 3.
- **C_length**: L0 F18861 (−5.03), L2 F23548 (−3.98), L2 F21401 (−3.51), L0 F9440 (−2.67). Dominated by very-early-layer features.
- **C_code**: small magnitudes, no dominant structure.

## Section 4 — Feature interpretation (logit lens over decoder direction)

- **C_syco** top features promote answer/correctness-related tokens: ` answers`,
  ` spelling`, and fluent suffixes (`-eous`, `-ibly`, `-fully`). A "producing an answer
  / correctness-judgement" register — plausibly where the model decides whether to
  endorse the user's stated answer.
- **C_length** top features promote **boilerplate and filler**: `Advertisement`,
  `Meanwhile`, `SPONSORED`, long whitespace runs, and punctuation (`.`, `,`, `;`) — the
  vocabulary of padding and verbose continuation, active in the earliest layers.
- **C_code** features promote code-surface tokens (`def`, `print`, `python`, `!=`,
  `==`) — GPT-2 has code *surface* features even though it cannot prefer exploits.

Sycophancy (answer/correctness register) and length (boilerplate/filler) feature
families are **semantically disjoint**.

---

## Section 5 — Initial observations

1. **Layer profiles are cleanly separated** (`results/figures/layer_distributions.png`):
   length bias lives in the **earliest layers (0, 2)** — a surface/stylistic decision;
   sycophancy lives in **mid-late layers (6–8)** — a higher-level judgement about whether
   to endorse the user. This matches intuition and is sharper than on synthetic data.
2. **The top syco features by magnitude are hacking-*resisting* (negative IE).** GPT-2
   has strong features pushing it to give the correct answer; sycophancy wins on average
   anyway (m=+1.23). This drives the Phase-3 ablation result.
3. Length's m_full is small (+0.11) — the behaviour is weak in GPT-2, so its circuit is
   noisier.

---

## Section 6 — Honest limitations

- **Faithfulness (mean-ablation) is only partly reliable.** For **syco** it is now
  non-degenerate (all 60 pairs stable, faith 0.09 — low but real). For **length** the
  long UltraFeedback completions still cause numerical blow-up (43/60 pairs NaN-skipped)
  and m_full is tiny, so its faithfulness (1.52) is not interpretable. The **discovered
  circuits (nodes/IE/edges/semantics)** are the reliable output; quantitative
  faithfulness is inconclusive on length. Phase 3 provides a **cleaner causal test**
  (held-out ablation vs random control) that does not depend on this metric.
- **Length bias is weak in GPT-2 Small** (m=+0.11, 58 %) — its circuit is the noisier of
  the two.
- **Code is a null** (no preference; real labelled data gated) — excluded from conclusions.
- **Residual-stream only; reduced scale; SAE quality not re-measured locally.**
